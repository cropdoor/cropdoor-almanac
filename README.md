# CropDoor Almanac

Architecture and reference docs for the CropDoor platform. Published at <https://almanac.cropdoor.com>.

The almanac is the trusted reference book for the CropDoor farm. It sits alongside the other agricultural-themed environments:

| Stage | Where |
| ----- | ----- |
| Testing | `nursery.cropdoor.com` |
| Staging | `field.cropdoor.com` |
| Reference docs | `almanac.cropdoor.com` |

## What this repo is

A MkDocs Material site that documents:

- The overall platform architecture
- Security, RBAC, audit logging
- Authentication and onboarding flows
- Domain models — farms, buyers, members, listings, orders, payments
- Notifications (email / SMS / voice)
- Operations and runbooks

Content is **net-new**, rooted in the actual code in `cropdoor-backend` (and later `cropdoor-frontend`). The pre-existing `cropdoor-backend/docs/` directory is a reference, not a source.

## What this repo is NOT

- Not the source of API specs — those come from the OpenAPI generated by Springdoc, surfaced separately.
- Not a CMS or blog. Every page is committed markdown.
- Not versioned (single "latest" view). Revisit if/when the platform needs multi-version docs.

## Develop locally

Requires Python 3.12+ and `pip`.

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

mkdocs serve
# → http://127.0.0.1:8000
```

Live reload on save. Build a one-off static site with `mkdocs build` (output in `site/`, gitignored).

## Deploy

Push to `main`. The `deploy` workflow in `.github/workflows/deploy.yml` builds the site, uploads it as a Pages artifact, and the deploy job publishes it to <https://almanac.cropdoor.com>. The custom domain is mapped via `docs/CNAME`.
